Getting somewhere.

JSON and CORS

Safewalk uses Firebase as their databse (I think). The website uses fetch() to read and write JSON data from the server. On my quest to contribute something, I found that the "Delete" buttons scattered throughout the site actually didn't work. After a few hours of tracking it down, I found that the delete fetch() code was formatted correctly, however delete is considered an unsafe operation by the server, and thus required a special option (Access-Control-Allow-Origin) in CORS (Cross-Origin Resource Sharing) to be enabled. After the last meeting, it was enabled and deleting users now works!

React-Google-Maps

Safewalk uses a react extension called react-google-maps in order to interface with the google maps api and display one of those giant page to page google maps you see everywhere. The idea was to display both walks and officers on the page and be able to click them for more options and info. I have just finished writing the code to show a little info window (Using the InfoWindow component) to show this. It includes modern ammenities like dashes in phone numbers, and colored status text. Truely modern. More work will have to be done in order for the Assign button to work from the map though, as it was built to work in a table.
